# 🚀 Инструкция по развертыванию Sonata-Крым на Beget

## 📋 Требования
- Хостинг Beget с поддержкой Node.js
- Домен: sonata.tagir75.ru (поддомен уже настроен)
- SSH-доступ к серверу

## 🔧 Шаг 1: Подготовка проекта локально

### 1.1 Установка зависимостей
```bash
cd /workspace/sonata-app
npm install
```

### 1.2 Инициализация базы данных
```bash
# Создание базы данных SQLite
npx prisma db push

# Заполнение данными из import_data.json
npx ts-node --compiler-options '{"module":"CommonJS"}' prisma/seed.ts
```

### 1.3 Тестирование локально
```bash
npm run dev
```
Откройте http://localhost:3000

## 📦 Шаг 2: Сборка проекта

```bash
npm run build
```

После сборки будет создана папка `.next` и standalone-версия в `.next/standalone`

## ☁️ Шаг 3: Загрузка на Beget

### 3.1 Подключение по SSH
```bash
ssh ваш-логин@beget.com
```

### 3.2 Переход в директорию сайта
```bash
cd /home/{ваш-логин}/sonata.tagir75.ru/
```

### 3.3 Загрузка файлов
Загрузите следующие файлы и папки:
- `package.json`
- `package-lock.json`
- `.next/standalone/` (всю папку)
- `.next/static/` (всю папку)
- `prisma/dev.db` (база данных)
- `public/logo.png`

**Важно:** Не загружайте всю папку `.next`, только `standalone` и `static`!

### 3.4 Установка зависимостей на сервере
```bash
cd /home/{ваш-логин}/sonata.tagir75.ru/
npm install --production
```

## ⚙️ Шаг 4: Настройка Node.js на Beget

### 4.1 В панели управления Beget:
1. Зайдите в панель управления хостингом
2. Перейдите в раздел "Node.js"
3. Выберите домен: `sonata.tagir75.ru`
4. Укажите версию Node.js: **18.x** или **20.x**
5. Путь к приложению: `/home/{ваш-логин}/sonata.tagir75.ru/`
6. Точка входа: `server.js` (создайте этот файл)

### 4.2 Создайте файл server.js в корне проекта:
```javascript
const { createServer } = require('http')
const next = require('next')

const port = parseInt(process.env.PORT || '3000', 10)
const dev = process.env.NODE_ENV !== 'production'
const app = next({ dev, dir: '.' })
const handle = app.getRequestHandler()

app.prepare().then(() => {
  createServer((req, res) => {
    handle(req, res)
  }).listen(port, () => {
    console.log(`> Ready on http://localhost:${port}`)
  })
})
```

## 🗄️ Шаг 5: База данных

База данных SQLite находится в файле `prisma/dev.db`. Она автоматически создается при первом запуске.

### Для обновления структуры БД:
```bash
npx prisma db push
```

### Для повторного заполнения данными:
```bash
npx ts-node --compiler-options '{"module":"CommonJS"}' prisma/seed.ts
```

## 🔧 Решение проблем

### Ошибка "No space left on device"
Очистите место на диске или удалите ненужные файлы:
```bash
rm -rf node_modules/.cache
rm -rf .next/cache
```

### Ошибка подключения к БД
Убедитесь, что файл `prisma/dev.db` существует и имеет права на запись:
```bash
chmod 644 prisma/dev.db
```

### Приложение не запускается
Проверьте логи Node.js в панели Beget или через:
```bash
tail -f ~/logs/nodejs.log
```

При возникновении проблем:

2. Убедитесь, что все зависимости установлены
3. Проверьте права доступа к файлам
4. Убедитесь, что порт 3000 свободен
